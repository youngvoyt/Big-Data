3. Выполнена интерпретация коэффициентов модели и сформулирован бизнес-вывод о связи стабильности темпа с полом пользователя.

### Почему добавлен уменьшенный файл

Полный `endomondoHR.json` занимает более 6 ГБ и плохо подходит для запуска в Google Colab.  
Поэтому для варианта 10 подготовлен компактный набор:

- `data/endomondoHR_sample_v10.json`
- `data/endomondoHR_sample_v10.json.gz`

В выборку включены только записи:

- с непустым массивом `speed`
- только для `gender in ('male', 'female')`
- все доступные женские записи
- детерминированная подвыборка мужских записей

Это уменьшает размер данных примерно до 100 МБ в архивированном виде и сохраняет пригодность датасета для демонстрации полного Spark ML pipeline.

## Что делает ноутбук

Ноутбук `lab_03_ml.ipynb` разбит на блоки:

1. `Setup`  
   Установка библиотек и импорт модулей.
2. `ETL`  
   Загрузка JSON, проверка схемы, фильтрация пропусков, первичный анализ.
3. `Feature Engineering`  
   Построение признаков `avg_speed`, `stddev_speed`, `speed_len`.
4. `ML Pipeline`  
   `StringIndexer -> VectorAssembler -> StandardScaler -> LogisticRegression`.
5. `Evaluation`  
   Расчет `ROC-AUC`, `Accuracy`, матрицы ошибок.
6. `Business Interpretation`  
   Анализ коэффициентов и вывод для продуктовой команды.

## Ключевая идея модели

Если коэффициент при `stddev_speed` положителен, модель связывает высокую вариативность темпа с большей вероятностью одного класса.  
Если отрицателен, более стабильный темп работает в сторону противоположного класса.

Такую модель можно использовать для:

- сегментации пользователей
- персонализации рекламных предложений
- уточнения профиля пользователя при неполных регистрационных данных
- повышения качества рекомендаций тренировок
