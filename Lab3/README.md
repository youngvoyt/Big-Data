## Описание работы

В данной лабораторной работе реализован полный ML-пайплайн на платформе **Apache Spark** с использованием библиотеки **Spark MLlib**.  
Работа выполнена на основе бизнес-кейса компании **HealthTech**, которая развивает цифровой фитнес-продукт по аналогии с Endomondo или Strava.

### Бизнес-задача

Предположим, что компания хочет лучше понимать поведение пользователей на основе данных о тренировках.  
В рамках **варианта 10** требуется построить модель, которая по характеристикам тренировки предсказывает **пол пользователя (`gender`)**.

Это позволяет:

- улучшать сегментацию аудитории;
- повышать качество персонализации;
- уточнять пользовательский профиль при неполных регистрационных данных;
- делать таргетированные предложения спортивных товаров и тренировочных программ.

## Задание варианта 10

### Задание 1. Подготовка данных и признаков

Необходимо построить признаки:

- `avg_speed` — средняя скорость тренировки;
- `stddev_speed` — стандартное отклонение скорости, характеризующее стабильность темпа.

Целевая переменная:

- `gender`

### Задание 2. Моделирование

Построить модель:

- `LogisticRegression`

с регуляризацией:

- `ElasticNet`

### Задание 3. Бизнес-интерпретация

Интерпретировать коэффициенты модели и ответить на вопрос:

**Как стабильность темпа связана с полом пользователя?**

## Что реализовано в проекте

В репозитории подготовлено полноценное решение, включающее:

## Используемые данные

**Датасет:** `endomondoHR.json`  
**Тематика:** данные о тренировках пользователей фитнес-приложения  
**Поля:** массивы скорости, пульса, высоты, временных меток, а также служебные атрибуты тренировки

### Почему в репозитории лежит уменьшенный датасет

Полный исходный файл `endomondoHR.json` имеет размер более **6 ГБ**, поэтому он плохо подходит для запуска в стандартной сессии **Google Colab**.

Для того чтобы сохранить воспроизводимость работы, была подготовлена облегченная версия:

- `data/endomondoHR_sample_v10.json`
- `data/endomondoHR_sample_v10.json.gz`

### Принцип формирования sample-набора

В компактный набор включены:

- записи только с `gender in ('male', 'female')`;
- записи только с непустым массивом `speed`;
- все доступные записи класса `female`;
- детерминированная подвыборка записей класса `male`.

Такой подход позволяет:

- существенно уменьшить размер данных;
- сохранить пригодность набора для построения ML-пайплайна;
- запускать решение в Colab без перегрузки памяти.

## Логика ноутбука

Ноутбук [lab_03_ml.ipynb](./lab_03_ml.ipynb) разбит на логические этапы:

1. `Setup`  
   Установка зависимостей и подготовка окружения Colab / Jupyter.

2. `Загрузка данных`  
   Поиск файла, ручная загрузка через Colab при необходимости, чтение JSON в Spark.

3. `ETL`  
   Анализ схемы, проверка пропусков, фильтрация невалидных наблюдений.

4. `Feature Engineering`  
   Расчет признаков `avg_speed`, `stddev_speed`, `speed_len`.

5. `ML Pipeline`  
   Построение конвейера:
   `StringIndexer -> VectorAssembler -> StandardScaler -> LogisticRegression`

6. `Evaluation`  
   Расчет метрик `ROC-AUC`, `Accuracy`, построение ROC-кривой и матрицы ошибок.

7. `Interpretation`  
   Анализ коэффициентов логистической регрессии.

8. `Business`  
   Формулировка бизнес-вывода по результатам модели.
